API란?

프로그램들이 서로 소통하는 방법을 알아보세요

📱 앱 ⇄ 🔌 API ⇄ 🖥️ 서버

API는 식당의 웨이터와 같습니다. 손님(클라이언트)이 메뉴(요청)를 주문하면, 웨이터(API)가 주방(서버)에 전달하고, 요리(응답)를 가져다줍니다.

HTTP 메서드

GET
데이터 조회
서버에서 데이터를 가져옵니다. 예: 게시글 목록 조회
POST
데이터 생성
서버에 새 데이터를 보냅니다. 예: 새 게시글 작성
PUT
데이터 수정
기존 데이터를 업데이트합니다. 예: 프로필 수정
DELETE
데이터 삭제
서버에서 데이터를 삭제합니다. 예: 댓글 삭제

날씨 API 체험

가상 날씨 API

GET/api/weather?city=서울
요청 (Request)
응답 (Response) - Status: 200 OK

주요 헤더

요청/응답에 포함되는 메타데이터

Content-Type데이터 형식 (JSON, HTML 등)
Authorization인증 토큰 (API 키 등)
Accept원하는 응답 형식
Cache-Control캐시 정책
User-Agent클라이언트 정보

API (Application Programming Interface)

API는 서로 다른 소프트웨어가 정해진 규칙에 따라 데이터를 주고받을 수 있게 해주는 인터페이스입니다. REST API는 HTTP 프로토콜을 사용하며, URL(엔드포인트)로 리소스를 지정하고, HTTP 메서드로 작업 종류를 구분합니다. 요청/응답 데이터는 주로 JSON 형식을 사용합니다.

API 동작 원리

API(Application Programming Interface)의 개념과 REST API의 동작 원리를 설명합니다. 엔드포인트, HTTP 메서드, JSON 데이터 형식 등을 이해할 수 있습니다.

사용 방법

API 요청을 시뮬레이션하며 요청/응답 과정을 확인하세요.

팁

REST API는 URL(엔드포인트) + HTTP 메서드(GET/POST/PUT/DELETE)로 자원을 조작합니다.
