📝 Git 기초

Git = 코드의 타임머신

파일의 변경 이력을 기록하고, 언제든 과거로 돌아갈 수 있는 버전 관리 시스템입니다.

기본 흐름

📁 작업 디렉토리파일 수정
→
📋 스테이징변경 선택
→
💾 저장소커밋 저장
→
☁️ 원격GitHub

필수 명령어

$ git init # 저장소 초기화
$ git add . # 변경 파일 스테이징
$ git commit -m "메시지" # 커밋 저장
$ git push origin main # 원격에 업로드
$ git pull # 원격에서 다운로드
$ git branch feature # 브랜치 생성
$ git merge feature # 브랜치 병합
$ git log --oneline # 커밋 이력 보기

🌿 브랜치란?

독립된 작업 공간을 만드는 기능입니다.

main ─── 메인 브랜치 (안정 버전)
feature ─── 새 기능 개발용
hotfix ─── 긴급 버그 수정용

작업 완료 후 merge로 합칩니다.
