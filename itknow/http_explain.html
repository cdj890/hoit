🌐 HTTP 동작 원리

각 단계를 클릭하면 상세 설명을 볼 수 있어요

1️⃣ URL 입력

브라우저 주소창에 URL을 입력합니다.
예: https://www.example.com/page

URL 구조: 프로토콜://도메인/경로

↓

2️⃣ DNS 조회

도메인 이름(www.example.com)을 IP 주소(93.184.216.34)로 변환합니다.

브라우저 캐시 → OS 캐시 → DNS 서버 순서로 조회합니다.

↓

3️⃣ TCP 연결

서버와 TCP 3-way 핸드셰이크로 연결을 수립합니다.

1. 클라이언트 → SYN
2. 서버 → SYN+ACK
3. 클라이언트 → ACK

HTTPS의 경우 TLS 핸드셰이크도 추가됩니다.

↓

4️⃣ HTTP 요청

브라우저가 서버에 HTTP 요청을 보냅니다.

GET /page HTTP/1.1
Host: www.example.com
Accept: text/html
User-Agent: Chrome/...

↓

5️⃣ 서버 처리

웹 서버가 요청을 처리합니다.

• 정적 파일 → 바로 반환
• 동적 페이지 → 서버 로직 실행 → DB 조회 → HTML 생성

↓

6️⃣ HTTP 응답

서버가 결과를 클라이언트에게 반환합니다.

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...페이지 내용...</html>

↓

7️⃣ 브라우저 렌더링

브라우저가 HTML을 파싱하여 화면에 표시합니다.

HTML 파싱 → DOM 트리 → CSS 파싱 → 렌더 트리 → 레이아웃 → 페인트
