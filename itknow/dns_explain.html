🔍 DNS 동작 원리

www.google.com 입력 → 142.250.196.100 반환

1. 브라우저 캐시 확인

이전에 방문한 적 있으면 브라우저가 IP를 기억하고 있어요

캐시에 없으면 ↓

2. OS 캐시 확인

운영체제의 DNS 캐시와 hosts 파일을 확인합니다

없으면 ↓

3. 로컬 DNS 서버 (ISP)

인터넷 서비스 제공자(KT, SKT 등)의 DNS 서버에 질의합니다

없으면 ↓

4. 루트 DNS 서버

전 세계 13개의 루트 서버 중 하나에 질의
".com은 어디서 관리하나요?"

↓ ".com은 여기서 관리해요"

5. TLD DNS 서버 (.com)

.com 도메인을 관리하는 서버에 질의
"google.com은 어디서 관리하나요?"

↓ "google.com은 여기서 관리해요"

6. 권한 DNS 서버 (google)

google.com을 실제로 관리하는 서버
"www.google.com의 IP는 142.250.196.100입니다!"

↓ IP 주소 반환!
✅ 142.250.196.100 으로 접속!

💡 알아두면 좋은 것들

• DNS 캐시 TTL: 보통 수 분~수 시간
• 공용 DNS: Google 8.8.8.8, Cloudflare 1.1.1.1
• DNS over HTTPS(DoH): 보안 강화된 DNS
• 전체 과정: 보통 수십 밀리초 이내
