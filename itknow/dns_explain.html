DNS 동작 원리

도메인 이름이 IP 주소로 변환되는 과정을 시각적으로 확인하세요

https://
도메인을 입력하고 조회 버튼을 눌러보세요
💻
1. 브라우저 캐시 확인
브라우저가 이전에 방문한 도메인의 IP를 캐시에서 먼저 찾습니다.
↓
🖥️
2. OS 캐시 / hosts 파일
운영체제의 DNS 캐시와 hosts 파일을 확인합니다.
↓
🏢
3. Local DNS 서버 (ISP)
인터넷 제공업체(ISP)의 DNS 서버에 질의합니다.
↓
🌍
4. Root DNS 서버
전 세계 13개 루트 서버 중 하나에 질의합니다. TLD 서버 주소를 알려줍니다.
↓
📂
5. TLD DNS 서버
.com, .kr 등 최상위 도메인 서버가 권한 네임서버 주소를 반환합니다.
↓
🎯
6. Authoritative DNS 서버
해당 도메인의 실제 DNS 레코드를 가진 서버가 최종 IP를 반환합니다.
↓
✅
7. IP 주소 반환
IP 주소가 브라우저에 전달되고, HTTP 연결을 시작합니다.

DNS란?

Domain Name System은 사람이 읽을 수 있는 도메인 이름(google.com)을 컴퓨터가 이해하는 IP 주소(142.250.196.110)로 변환하는 시스템입니다. 인터넷의 전화번호부라고 할 수 있습니다. 한 번 조회된 결과는 TTL(Time To Live) 동안 캐시에 저장됩니다.
